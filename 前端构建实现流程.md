# 前端构建实现流程

## 1. 文档目标

本文档用于指导本项目后续前端从 0 到 1 的建设，作为统一实施标准。核心目标：

- 基于现有后端接口能力构建完整前端
- 明确阶段划分、模块边界、实现顺序、验收标准
- 后续前端开发统一按本文档推进，并持续补充进度

---

## 2. 项目定位

当前仓库本质上是一个图片资源管理后端，包含以下业务域：

- 用户体系：登录、登出、获取当前用户、管理员用户管理
- 图片体系：上传、URL 导入、批量导入、编辑、删除、分页查询、详情、审核、标签分类、AI 扩图
- 空间体系：空间创建、编辑、删除、详情、分页查询
- 成员体系：空间成员添加、删除、查询、角色编辑、我的空间
- 协同体系：图片编辑 WebSocket，支持进入编辑、退出编辑、编辑动作广播

因此前端不是普通官网，而应定位为：

- 图片广场 + 图片工作台 + 空间协作平台 + 管理后台

---

## 3. 总体建设原则

后续前端建设遵循以下原则：

- 先打通主链路，再做高级体验
- 先做高频页面，再做增强能力
- 先做稳定的工程结构，再铺业务页面
- 先完成 MVP，再迭代协同编辑与 AI 能力
- 每完成一个阶段，及时回写本文件进度

---

## 4. 推荐技术选型

建议第一版采用以下技术栈：

- React 18
- TypeScript
- Vite
- React Router
- TanStack Query
- Zustand
- Axios
- Ant Design
- Tailwind CSS 或 SCSS Modules
- ECharts

选型原因：

- React + Vite 搭建快，联调效率高
- TypeScript 适合沉淀后端接口类型
- Query 适合服务端状态，如图片列表、空间详情、成员列表
- Zustand 适合维护登录用户、当前空间、界面状态
- Ant Design 适合后台、表格、表单、弹窗

---

## 5. 视觉与产品方向

建议前端整体风格采用“深色图片工作台”。

设计关键词：

- 深色背景
- 强调图片内容
- 卡片化布局
- 简洁但专业的工具感
- 后台高信息密度

页面风格区分：

- 主站：首页、图片广场、详情页，强调视觉体验
- 工作台：上传、空间、编辑页，强调效率与信息结构
- 后台：审核台、用户管理，强调密度与操作效率

---

## 6. 前端路由规划

建议规划以下一级路由：

- `/`：首页 / 图片广场
- `/login`：登录页
- `/register`：注册页
- `/picture/:id`：图片详情页
- `/upload`：上传中心
- `/spaces`：空间列表
- `/space/:id`：空间工作台
- `/space/:id/members`：空间成员管理
- `/editor/:pictureId`：协同编辑页
- `/admin/review`：图片审核台
- `/admin/users`：用户管理
- `/me`：个人中心

布局建议拆分为：

- `BasicLayout`
- `UserLayout`
- `WorkspaceLayout`
- `AdminLayout`

## 7. 功能模块拆分

### 7.1 用户模块

范围：

- 登录
- 登出
- 获取当前用户
- 个人中心
- 管理员用户管理

实现要求：

- 启动时拉取当前登录用户
- 使用路由守卫控制页面访问
- 使用角色守卫控制管理员入口

### 7.2 图片模块

范围：

- 图片分页查询
- 图片详情
- 本地上传
- URL 上传
- 批量导入
- 图片编辑
- 图片删除
- 标签分类展示
- AI 扩图

实现要求：

- 列表页支持关键词、标签、分类、审核状态、空间筛选
- 详情页展示图片信息、作者信息、空间信息、操作区
- 上传页支持拖拽、预览、重命名、选择空间
- AI 扩图采用任务式流程：创建任务 -> 轮询状态 -> 展示结果

### 7.3 空间模块

范围：

- 创建空间
- 编辑空间
- 删除空间
- 空间列表
- 空间详情
- 空间配额展示

实现要求：

- 展示空间类型、空间级别、已用容量、图片数量
- 空间详情页中展示该空间下图片列表
- 顶部固定显示空间概览信息

### 7.4 空间成员模块

范围：

- 添加成员
- 删除成员
- 查询成员
- 修改成员角色
- 我的空间列表

实现要求：

- 团队空间显示成员管理入口
- 角色映射统一使用中文显示：浏览者 / 编辑者 / 管理员
- 按角色控制按钮显隐

### 7.5 管理后台模块

范围：

- 图片审核
- 用户管理
- 基础数据看板

实现要求：

- 审核页支持筛选、分页、通过、驳回
- 用户管理页支持检索与分页
- 管理后台统一使用高密度布局

### 7.6 实时协同模块

范围：

- 进入编辑
- 退出编辑
- 在线用户提示
- 编辑动作广播

实现要求：

- 第一版不实现重型图片编辑器
- 第一版先实现在线状态、进入/退出提示、基础动作同步
- 后续再迭代真正的可视化编辑能力

## 9. 接口层实现规范

### 9.1 请求封装规范

统一封装请求层，处理：

- 基础地址 `/api`
- 超时处理
- 错误提示
- 响应解包
- 未登录处理
- 统一消息提示

### 9.2 类型规范

前端优先建立以下类型：

- `BaseResponse<T>`
- `LoginUser`
- `UserVO`
- `PictureVO`
- `SpaceVO`
- `SpaceUserVO`
- `PageResult<T>`

### 9.3 枚举映射规范

需在前端建立统一枚举映射：

- 用户角色：`user` / `admin`
- 空间类型：私有空间 / 团队空间
- 空间角色：`viewer` / `editor` / `admin`
- 审核状态：待审核 / 通过 / 驳回

## 10. 鉴权与权限设计

### 10.1 登录态策略

由于后端当前使用 Session 登录态，前端联调建议：

- 使用 Vite 代理访问后端 `/api`
- 避免浏览器跨域凭证问题
- 页面刷新后重新获取当前登录用户

### 10.2 权限分层

分三层控制：

#### 页面权限

- 未登录用户不能访问上传中心、空间页、个人中心、后台页
- 普通用户不能访问管理员后台页
- 管理员可访问全部页面

#### 按钮权限

- 仅管理员显示审核和用户管理操作
- 仅空间管理员显示成员管理入口
- 仅图片所有者或管理员显示删除、编辑等敏感操作

#### 数据权限提示

- 无权限时页面展示明确提示，不允许白屏
- 提供返回上一级或返回首页入口

## 11. 页面实施顺序

### 第一阶段：工程初始化

目标：搭建基础壳

任务：

- 初始化前端项目
- 配置路由
- 配置请求层
- 配置全局状态
- 配置主题变量
- 配置开发代理

交付：

- 前端工程可运行
- 可访问基础空页面
- 可连通健康检查接口

### 第二阶段：首页与用户链路

目标：让用户先能进入系统并浏览图片

任务：

- 登录页
- 注册页
- 顶部导航
- 图片广场
- 图片详情页

交付：

- 用户可以登录
- 用户可以查看图片列表与详情

### 第三阶段：上传中心

目标：打通核心生产能力

任务：

- 本地上传
- URL 上传
- 批量导入入口
- 上传完成反馈与跳转

交付：

- 用户完成图片导入闭环

### 第四阶段：空间工作台

目标：打通图片组织能力

任务：

- 空间列表
- 创建空间
- 空间详情
- 空间图片列表
- 空间配额展示

交付：

- 用户可在空间内管理图片

### 第五阶段：空间成员协作

目标：打通团队协作基础能力

任务：

- 成员列表
- 添加成员
- 编辑角色
- 删除成员
- 我的空间增强展示

交付：

- 团队空间可进行基础成员协作管理

### 第六阶段：管理员后台

目标：打通治理能力

任务：

- 图片审核台
- 用户管理页
- 管理后台筛选与表格能力

交付：

- 管理员可完成核心治理操作

### 第七阶段：协同编辑与 AI 扩图

目标：形成差异化能力

任务：

- WebSocket 接入
- 在线状态提示
- 基础编辑动作同步
- AI 扩图任务流转页
- 任务轮询与结果展示

交付：

- 具备基础协同能力
- 具备 AI 扩图使用闭环

## 13. 状态管理策略

建议分为三类：

### 服务端状态

由 TanStack Query 管理：

- 当前登录用户
- 图片列表
- 图片详情
- 空间列表
- 空间详情
- 成员列表
- 审核列表

### 全局客户端状态

由 Zustand 管理：

- 当前空间上下文
- 当前主题
- 弹窗开关
- 当前筛选器草稿

### 页面局部状态

由组件本地管理：

- 表单输入
- 当前 Tab
- 上传进度
- 临时预览状态

## 14. 联调流程

建议联调顺序如下：

1. 健康检查联通
2. 登录 / 当前用户联通
3. 图片列表与详情联通
4. 上传能力联通
5. 空间能力联通
6. 成员管理联通
7. 管理后台联通
8. WebSocket 联通
9. AI 扩图联通

联调重点风险：

- Session 登录态在跨域场景下容易出问题
- 上传接口分本地文件、URL、批量导入三种路径
- 批量导入与 AI 扩图是耗时流程，需要前端处理异步状态
- 某些接口有管理员权限限制，需要提前准备测试账号

---

## 15. 验收标准

### MVP 验收标准

达到以下条件即可视为第一版前端完成：

- 用户可登录并获取当前用户信息
- 用户可浏览图片列表与详情
- 用户可完成本地上传和 URL 上传
- 用户可查看空间列表和空间详情
- 管理员可进入审核台并完成审核

### 二期验收标准

- 可完成团队空间成员管理
- 可展示在线编辑状态
- 可完成 AI 扩图全流程

### 工程验收标准

- 目录结构清晰
- 接口层封装统一
- 样式变量统一管理
- 页面具备加载态、空态、错误态
- 权限控制清晰，不出现越权按钮

---

1. 创建前端项目骨架

3. 完成登录页、首页、详情页
4. 完成上传中心
5. 完成空间工作台
6. 完成成员管理
7. 完成审核后台
8. 完成协同编辑与 AI 扩图

每完成一个阶段，需要同步更新本文档中的：

- 已完成内容
- 当前问题
- 下一阶段计划

使本文档持续作为前端实施主文档使用。
